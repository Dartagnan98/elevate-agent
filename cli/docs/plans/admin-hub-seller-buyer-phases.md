# Admin Hub — Seller and Buyer Workflow Phases

## Core Boundary

The Admin Hub should not start at raw lead capture.

It should start when the client/opportunity is already real enough to require operations, paperwork, deadlines, coordination, and follow-up.

**Seller start point:** after the CMA stage.

**Buyer start point:** after the buyer agrees to do walkthroughs/showings.

Everything before that belongs in separate lead generation, nurture, qualification, and outreach lanes.

---

# Seller Workflow

## Trigger

The seller workflow starts when:

- CMA is complete or mostly complete.
- Seller is likely moving forward.
- Listing conversation is active.
- pilot realtor needs operational support to move from pricing conversation to signed listing.

It does **not** start at:

- Random seller lead.
- Home valuation request.
- Cold nurture.
- General “thinking of selling someday” contact.

## Seller Flow

```text
CMA complete
→ Post-CMA Seller Conversion
→ Listing Intake
→ MLC / Listing Paperwork
→ Listing Launch
→ Active Listing Management
→ Offer Management
→ Accepted Offer / Transaction Heartbeat
→ Completion
→ Post-close
```

---

## Phase 1 — Post-CMA Seller Conversion

**Goal:** move from CMA/pricing conversation to a signed listing path.

AI/admin support:

- Draft CMA follow-up message.
- Summarize pricing recommendation.
- Prepare seller next-step checklist.
- Track seller questions and objections.
- Identify missing information needed before listing paperwork.
- Prepare listing intake request.

Outputs:

- Seller follow-up draft.
- Pricing recap.
- Next-step checklist.
- Missing-info list.

---

## Phase 2 — Listing Intake

**Goal:** collect everything needed to prepare the listing package and launch plan.

Information needed:

- Seller legal names.
- Property address.
- Title status.
- Listing price.
- Commission.
- Listing start/end dates.
- Included items.
- Excluded items.
- Possession preferences.
- Tenancy details, if any.
- Strata documents, if applicable.
- Showing instructions.
- Lockbox details.
- Pets/access restrictions.
- Photo/video/floorplan timing.
- Preferred communication method.

AI/admin support:

- Create listing intake checklist.
- Identify missing fields.
- Prepare seller questions.
- Start transaction/listing folder.
- Prepare data for MLC/forms.

Outputs:

- Complete listing intake record.
- Missing-info checklist.
- Seller question draft.
- Listing folder/task setup.

---

## Phase 3 — MLC / Listing Paperwork

**Goal:** get the listing paperwork completed, signed, and stored.

AI/admin support:

- Fill Multiple Listing Contract inputs.
- Prepare required listing forms.
- Check for missing signatures/initials.
- Send via DigiSign when approved.
- Track signature status.
- Save completed documents to the correct folder.
- Record envelope/status history.

Outputs:

- Filled listing forms.
- DigiSign envelope/status.
- Signed listing package.
- Stored document record.

---

## Phase 4 — Listing Launch

**Goal:** get the property live and marketed without delay.

AI/admin support:

- Draft MLS remarks.
- Draft public description.
- Draft feature sheet copy.
- Coordinate photo/video/floorplan checklist.
- Build social captions.
- Build email campaign copy.
- Build ad copy.
- Prepare launch checklist.

Outputs:

- MLS copy.
- Marketing copy.
- Social posts.
- Email campaign draft.
- Launch checklist.

---

## Phase 5 — Active Listing Management

**Goal:** keep the seller informed and push the listing toward an offer.

Inputs:

- ShowingTime feedback.
- Agent feedback.
- Buyer comments.
- MLS activity.
- Online activity.
- New competing listings.
- Recent sales.
- Price changes in the market.

AI/admin support:

- Collect showing feedback.
- Summarize activity.
- Build weekly seller update.
- Flag pricing or marketing issues.
- Recommend action changes.
- Draft seller update email.

Outputs:

- Weekly seller update.
- Feedback summary.
- Market movement summary.
- Price/action recommendation.

---

## Phase 6 — Offer Management

**Goal:** help pilot realtor quickly understand and explain offer terms.

Offer details to track:

- Purchase price.
- Deposit.
- Subject/condition dates.
- Completion date.
- Possession date.
- Adjustment date.
- Included items.
- Excluded items.
- Financing condition.
- Inspection condition.
- Sale-of-buyer-property condition.
- Special clauses.

AI/admin support:

- Summarize offer terms.
- Compare offer to seller goals.
- Flag risks or unusual clauses.
- Prepare seller explanation.
- Draft counter strategy notes.
- Track offer/counter versions.

Outputs:

- Offer summary.
- Seller explanation.
- Risk notes.
- Counter strategy draft.

---

## Phase 7 — Accepted Offer / Transaction Heartbeat

**Goal:** make sure deadlines and obligations do not get missed.

Seller-side heartbeat items:

- Subject removal deadline.
- Deposit status.
- Inspection timing.
- Document delivery.
- Lawyer/conveyancer coordination.
- Completion date.
- Possession date.
- Key/access instructions.
- Seller move-out reminders.

AI/admin support:

- Track all critical dates.
- Create reminders.
- Request missing documents.
- Draft seller updates.
- Flag overdue or at-risk items.
- Keep transaction status current.

Outputs:

- Transaction heartbeat timeline.
- Deadline reminders.
- Missing-item list.
- Seller update drafts.

---

## Phase 8 — Completion / Post-Close

**Goal:** close cleanly and move the seller into referral/nurture.

AI/admin support:

- Prepare completion checklist.
- Confirm possession/key instructions.
- Draft thank-you message.
- Draft review request.
- Draft referral ask.
- Add home anniversary/follow-up reminders.
- Move seller into past-client nurture.

Outputs:

- Completion checklist.
- Thank-you/review/referral messages.
- Past-client nurture record.

---

# Buyer Workflow

## Trigger

The buyer workflow starts when:

- Buyer agrees to do walkthroughs/showings.
- Buyer is actively looking.
- Search/showing coordination is needed.
- pilot realtor needs operational support to manage viewings, feedback, offer prep, and transaction deadlines.

It does **not** start at:

- Random buyer inquiry.
- Website registration.
- Cold buyer lead.
- Unqualified browser.
- Early nurture contact.

## Buyer Flow

```text
Buyer agrees to walkthroughs/showings
→ Buyer Walkthrough Intake
→ Search + Showing Plan
→ Showing Follow-Up
→ Offer Prep
→ Offer / Negotiation
→ Accepted Offer / Subject Removal
→ Completion / Possession
→ Post-close
```

---

## Phase 1 — Buyer Walkthrough Intake

**Goal:** make sure showings are organized and worth pilot realtor’s time.

Information needed:

- Buyer legal names.
- Budget.
- Financing/pre-approval status.
- Desired areas.
- Property type.
- Bedrooms/bathrooms.
- Must-haves.
- Dealbreakers.
- Timeline.
- Availability.
- Current home status.
- Buyer agency/agent relationship status.
- Preferred communication method.

AI/admin support:

- Build buyer profile.
- Identify missing information.
- Confirm showing readiness.
- Draft buyer intake questions.
- Prepare showing coordination checklist.

Outputs:

- Buyer walkthrough profile.
- Missing-info list.
- Showing readiness status.
- Buyer questions draft.

---

## Phase 2 — Search + Showing Plan

**Goal:** organize the active search and showing route.

AI/admin support:

- Translate buyer needs into search criteria.
- Prepare MLS/Lofty search filters.
- Shortlist properties.
- Rank properties by fit.
- Build showing route.
- Coordinate availability.
- Prepare property preview notes.

Outputs:

- Buyer search criteria.
- Property shortlist.
- Showing itinerary.
- Preview notes.

---

## Phase 3 — Showing Follow-Up

**Goal:** keep momentum and sharpen the buyer’s criteria.

Feedback to capture:

- What they liked.
- What they disliked.
- Dealbreakers discovered.
- Price comfort.
- Area feedback.
- Layout/condition feedback.
- Offer interest.
- Criteria changes.

AI/admin support:

- Draft follow-up after each showing.
- Summarize buyer feedback.
- Update buyer preferences.
- Flag possible offer properties.
- Recommend next showings.

Outputs:

- Showing feedback summary.
- Updated search criteria.
- Next showing plan.
- Possible-offer flag.

---

## Phase 4 — Offer Prep

**Goal:** prepare a smart offer before writing.

Information needed:

- Target property.
- Buyer names.
- Offer price.
- Deposit amount.
- Financing condition.
- Inspection condition.
- Insurance condition.
- Strata document review, if applicable.
- Completion date.
- Possession date.
- Adjustment date.
- Included items.
- Special clauses.

AI/admin support:

- Pull property history.
- Summarize comparable sales.
- Identify offer risks.
- Prepare offer strategy notes.
- Prepare CPS input checklist.
- Draft buyer explanation.

Outputs:

- Offer prep brief.
- Comparable summary.
- CPS input checklist.
- Buyer explanation draft.

---

## Phase 5 — Offer / Negotiation

**Goal:** get the offer accepted while protecting the buyer.

AI/admin support:

- Track offer versions.
- Summarize counter terms.
- Explain risks.
- Flag unusual clauses.
- Prepare counter strategy notes.
- Track acceptance status.

Outputs:

- Offer/counter summary.
- Buyer explanation.
- Risk notes.
- Negotiation timeline.

---

## Phase 6 — Accepted Offer / Subject Removal

**Goal:** manage all buyer-side deadlines.

Buyer-side heartbeat items:

- Subject removal date.
- Inspection deadline.
- Financing deadline.
- Insurance deadline.
- Strata review deadline.
- Deposit due date.
- Lawyer/conveyancer info.
- Completion date.
- Possession date.

AI/admin support:

- Track deadlines.
- Create reminders.
- Coordinate inspection.
- Follow up on financing.
- Track deposit.
- Request missing buyer/lawyer information.
- Draft buyer updates.
- Flag overdue or at-risk items.

Outputs:

- Buyer transaction heartbeat.
- Subject removal checklist.
- Deadline reminders.
- Buyer update drafts.

---

## Phase 7 — Completion / Possession

**Goal:** get the buyer to keys without missed steps.

AI/admin support:

- Prepare completion checklist.
- Track lawyer/conveyancer status.
- Prepare possession reminders.
- Draft utility/change-of-address reminders.
- Coordinate final walkthrough if needed.
- Prepare key handoff checklist.

Outputs:

- Completion checklist.
- Possession checklist.
- Utility/move-in reminder draft.
- Key handoff notes.

---

## Phase 8 — Post-Close

**Goal:** turn the buyer into repeat/referral business.

AI/admin support:

- Draft thank-you message.
- Draft review request.
- Draft referral ask.
- Add home anniversary reminder.
- Move buyer into past-client nurture.

Outputs:

- Thank-you/review/referral messages.
- Past-client nurture record.
- Home anniversary reminder.

# Shared Admin Hub Concepts

## Transaction Heartbeat

Both seller and buyer workflows eventually feed into the same deadline engine.

The Transaction Heartbeat tracks:

- Critical dates.
- Subject/condition dates.
- Deposit deadlines.
- Inspection deadlines.
- Financing deadlines.
- Document requirements.
- Lawyer/conveyancer status.
- Completion.
- Possession.
- Human vs AI responsibility.

## Source of Truth

The Admin Hub should eventually connect to:

- Lofty.
- Gmail.
- Google Calendar.
- Google Drive.
- DigiSign.
- WEBForms / TransactionDesk.
- SkySlope.
- ShowingTime.
- MLS/AOIR data where available.
- eXp Agent Centre / Transaction Guide source snapshots.

## Separation From Lead Gen

The Admin Hub starts after real intent.

Upstream systems should handle:

- Lead capture.
- Nurture.
- Cold outreach.
- Early buyer qualification.
- Early seller valuation requests.
- CMA request handling before the CMA is completed.

Admin Hub should handle:

- Listing operations.
- Showing operations.
- Paperwork.
- Deadlines.
- Seller updates.
- Buyer updates.
- Transaction coordination.
- Post-close handoff.

# Recommended Build Order

## Build 1 — Seller Post-CMA to Listing Intake

```text
CMA complete → seller follow-up → intake checklist → missing info → MLC prep
```

Why first:

- Highest leverage.
- Closest to listings.
- Reduces admin drag after a seller says yes.

## Build 2 — MLC / Listing Paperwork Lane

```text
listing intake → forms → DigiSign → signed package → saved docs
```

Why second:

- Turns seller intent into a signed listing.
- Prevents missed fields and paperwork delays.

## Build 3 — Active Listing Updates

```text
ShowingTime + market activity → weekly seller update → action recommendation
```

Why third:

- Keeps sellers calm.
- Creates clear pricing/action conversations.

## Build 4 — Transaction Heartbeat

```text
accepted offer → subjects/deposit/docs/lawyer/completion/possession tracking
```

Why fourth:

- Shared by both seller and buyer workflows.
- Prevents missed deadlines.

## Build 5 — Buyer Walkthrough to Showing Plan

```text
buyer agrees to showings → intake → search criteria → showing route → follow-up
```

Why fifth:

- Useful once seller ops are stable.
- Protects pilot realtor’s time with buyers.

## Build 6 — Buyer Offer + Subject Removal

```text
possible offer → offer prep → accepted offer → subject removal heartbeat
```

Why sixth:

- Connects buyer workflow into Transaction Heartbeat.
- Helps with risk/deadline management.

---

# Product Principle

The Admin Hub is not a CRM replacement and not a lead-gen toy.

It is the operating layer after intent is real:

```text
Seller: after CMA
Buyer: after walkthrough/showing commitment
```

Its job is to make sure pilot realtor always knows:

- What stage each client is in.
- What is missing.
- What deadline is next.
- What needs human approval.
- What Elevate can draft or prepare.
- What should happen today.
