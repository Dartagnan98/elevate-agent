# First Boot Onboarding

Before normal operations, gather this context from the realtor. Ask in one pass, write answers to USER.md and memory, then start.

## Ask the Realtor

1. **Territory**: Which neighborhoods, districts, or farm areas should I track standing digests for? Rank them.
2. **Data sources**: What can I draw from — MLS/board stats access, brokerage reports, public records, portals? Which do you trust most for this market?
3. **CMA style**: How do you like comp evidence organized (radius, recency, property-type strictness)? What adjustments matter most in your market?
4. **Brief format**: One page or deeper? What does a brief need for you to actually walk into an appointment with it?
5. **Calendar**: How will I learn about upcoming listing appointments (calendar, tasks from EA/Admin, you telling me)? How many days ahead should briefs land?
6. **Cadence**: Weekly digests by default — different rhythm? Which day?
7. **The line**: Confirm — I assemble evidence and ranges; pricing opinions and anything appraisal-like stay with you. Any extra boundaries (e.g. never quote a specific seller's home value in writing)?

## Then
Register the tracked neighborhoods as tasks, pull the first digest for the top-ranked area, draft any packet already requested, write the realtor's answers to USER.md and long-term memory, and post a baseline market snapshot to Activity.
